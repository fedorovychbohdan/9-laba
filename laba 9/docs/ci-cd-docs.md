# Автоматизація документації (CI/CD Pipeline)

Цей документ описує архітектуру та процес автоматичної збірки та публікації документації для системи бронювання квитків.

## 1. Артефакти документації
Автоматичній генерації підлягають наступні компоненти:

* **Загальний сайт документації:** Статичний сайт, зібраний за допомогою MkDocs (основа проекту).
* **API-документація (Swagger UI):** Інтерактивна специфікація, що генерується на основі `openapi.yaml`.
* **UI-документація (Storybook):** Візуальний гайд по компонентах інтерфейсу.

## 2. Тригери генерації (Triggers)
Процес збірки запускається автоматично за наступних умов:

* **Push у feature-гілки:** Запуск валідації та збірки (без публікації) для перевірки цілісності документації.
* **Merge у гілку `main`:** Повний цикл збірки та оновлення публічного сайту на GitHub Pages.

## 3. Послідовність кроків (Pipeline Steps)
Пайплайн складається з трьох основних етапів:

1. **Валідація (Validation):** 
    * Linting Markdown-файлів на наявність битих посилань.
    * Schema validation для файлу `openapi.yaml` (перевірка на відповідність стандарту OpenAPI 3.0).
2. **Збірка (Build):**
    * Генерація статичних HTML-сторінок через `mkdocs build`.
    * Збірка статики Storybook.
3. **Публікація (Deployment):**
    * Перенесення зібраних артефактів у гілку `gh-pages`.

## 4. Місце публікації та структура URL
* **Хостинг:** GitHub Pages.
* **Базова адреса:** `https://github.com/fedorovychbohdan/9-laba`.
* **Структура:**
    * `/` — Головна сторінка документації.
    * `/swagger/` — Інтерактивний API-контракт.
    * `/ui-components/` — Документація UI-компонентів.

## 5. Політика версіонування (Versioning)
Ми використовуємо підхід семантичного версіонування (SemVer):

* **Синхронізація:** Версія документації завжди відповідає версії продукту (наприклад, v1.2.0).
* **Правила оновлення:** 

    * **Major:** При зворотно-несумісних змінах API (Breaking changes).
    * **Minor:** При додаванні нових ендпоінтів або UI-компонентів.
    * **Patch:** При виправленні помилок у текстах або документації.
