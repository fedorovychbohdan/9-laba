# Documentation Overview: REST API сервіс бронювання квитків

Даний репозиторій містить повний пакет технічної документації для системи бронювання квитків, реалізований за підходом **Documentation as Code (DaC)**.

## Про проект
Система призначена для автоматизації продажу квитків, керування інвентарем місць у реальному часі та інтеграції з платіжними системами. Архітектура побудована на мікросервісах для забезпечення високої відмовостійкості та масштабованості.

## Структура документації

Для швидкої навігації використовуйте посилання нижче:

### Архітектура та вимоги
- [System Specification (SSD)](docs/architecture/SSD.md) — єдине джерело істини для функціональних та нефункціональних вимог.
- [Software Design (SDD)](docs/architecture/SDD.md) — опис мікросервісної архітектури, компонентів та API ендпоінтів.
- [Infrastructure (ISD)](docs/architecture/ISD.md) — специфікація хмарного середовища, Kubernetes та стратегій масштабування.

### Процеси та стандарти
- [Single Source of Truth (SSoT)](docs/process/ssot.md) — визначення головних документів та правил оновлення документації.
- [Software Maintenance Manual](docs/process/maintenance.md) — інструкції з підтримки та оновлення API без зупинки сервісу.
- [Project Roadmap](docs/process/roadmap.md) — графік розробки функцій та розподіл ресурсів команди.

### Якість та тестування
- [Test Strategy](docs/quality/test-strategy.md) — стратегія багаторівневої перевірки якості (Unit, Integration, Load).
- [Test Plan](docs/quality/test-plan.md) — детальні сценарії тестування критичних функцій системи.
- [Traceability Matrix](docs/quality/traceability-matrix.md) — матриця простежуваності між вимогами SSD та тест-кейсами.
