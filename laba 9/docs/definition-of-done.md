# Definition of Done (Documentation)

Цей документ визначає критерії готовності (DoD) для будь-яких змін у системі бронювання квитків з точки зору документації. Завдання вважається виконаним лише за умови виконання наведених нижче пунктів.

## 1. Матриця перевірки змін

| Тип зміни | Що оновити (Артефакти) | Перевірка (Validation) |
| :--- | :--- | :--- |
| **Новий API ендпоїнт** | `openapi.yaml`, `api-contract.md` | OpenAPI Schema Validation (Swagger) |
| **Зміна логіки API** | Опис параметрів у `openapi.yaml` | Тест запиту в Swagger UI (Try it out) |
| **Новий UI-компонент** | Додати Story в `ui-components.md` | Успішна збірка сайту (MkDocs build) |
| **Зміна дизайну UI** | Оновити стани (Default/Error) у Stories | Візуальна відповідність опису |
| **Нефункціональні зміни** | `ci-cd-docs.md` (якщо змінився пайплайн) | Перевірка цілісності внутрішніх посилань |

## 2. Критерії готовності (Checklist)

Щоб зміна вважалася "Done", вона має відповідати наступним вимогам:

* **Існування:** Для кожного нового методу API створено запис у специфікації.
* **Формат:** Специфікація відповідає стандарту OpenAPI 3.0 (YAML).
* **Доступність:** Нова документація відображається у відповідному розділі сайту.
* **Автоматизація:** Сайт успішно проходить процес `gh-deploy` без помилок збірки.

## 3. Місце перевірки
Всі перевірки проводяться локально перед коммітом за допомогою команди `mkdocs serve` та автоматично при пуші в репозиторій.
