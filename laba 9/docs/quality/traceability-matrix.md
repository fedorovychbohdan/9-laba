# Traceability Matrix (Матриця простежуваності) — v1.0
> **Див. також:** Ця матриця пов'язує тести з ідентифікаторами вимог, що описані в [SSD](../architecture/SSD.md).

Цей документ пов’язує функціональні та нефункціональні вимоги зі специфікації [SSD](../architecture/SSD.md) з відповідними тест-кейсами. Це гарантує повне покриття системи тестами та відсутність неперевіреного функціоналу.

| Вимога (ID) | Опис вимоги (згідно з SSD) | Тест-кейс (ID) | Опис тесту |
| :--- | :--- | :--- | :--- |
| **FR-01** | Пошук подій за фільтрами | **TC-01** | Перевірка `GET /events` з різними параметрами (категорія, дата). |
| **FR-02** | Створення тимчасового бронювання | **TC-02** | Створення броні через `POST /bookings` та перевірка статусу "Locked" у БД. |
| **FR-03** | Автоматичне скасування (TTL) | **TC-03** | Перевірка звільнення місця, якщо оплата не надійшла протягом 15 хвилин. |
| **FR-04** | Підтвердження оплати | **TC-04** | Емуляція успішного колбеку від Stripe та зміна статусу замовлення на "Paid". |
| **FR-05** | Генерація електронного квитка | **TC-05** | Перевірка створення PDF-файлу та його наявності в Object Storage (S3). |
| **NFR-01** | Продуктивність (1000 зап/хв) | **TC-06** | Навантажувальне тестування (Stress Test) тривалістю 30 хв. |

## Статус покриття
- **Функціональні вимоги (FR):** 100% покриття.
- **Нефункціональні вимоги (NFR):** 100% покриття.
