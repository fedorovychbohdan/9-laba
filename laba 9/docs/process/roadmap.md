# Project Plan / Roadmap — v1.0
> **Див. також:** Цей план відображає етапи реалізації функціональних вимог, детально описаних у [Специфікації системи (SSD)](../architecture/SSD.md).

## 1. План розробки (Milestones)
Процес створення системи розділений на чотири основні етапи:

| Етап | Назва | Ключові функції | Статус |
| :--- | :--- | :--- | :--- |
| **M1** | Core API | Базова інфраструктура, авторизація, каталог подій. | Done |
| **M2** | Booking Engine | Логіка бронювання, робота з Redis (TTL-lock). | In Progress |
| **M3** | Payment & Fiscal | Інтеграція зі Stripe/LiqPay, генерація квитків. | To Do |
| **M4** | Optimization | Навантажувальне тестування, впровадження CDN. | To Do |

## 2. Розподіл ресурсів команди
* **Backend Team (2 розробники):** Розробка мікросервісів, API Gateway та інтеграція БД.
* **QA Team (1 інженер):** Написання тестів (Unit, Integration, Load) та ведення Матриці простежуваності.
* **DevOps (1 інженер):** Налаштування Kubernetes-кластера, CI/CD пайплайнів та моніторингу.

## 3. Ризики та припущення
- **Технічні ризики:** Затримки в роботі зовнішніх платіжних систем (Stripe).
- **Мінімальні вимоги:** Система має бути готова до масштабування до 1000 зап/хв до завершення етапу M4.
