    Software Maintenance Manual — v1.0

    Див. також: Процедури підтримки базуються на мікросервісній структурі та специфікаціях API, описаних у Технічному дизайні (SDD).

    1. Мета документа

    Цей посібник визначає процедури технічної підтримки, оновлення та управління життєвим циклом системи бронювання квитків для забезпечення її безперебійної роботи.

    2. Оновлення API без зупинки сервісу (Zero Downtime Deployment)

    Для внесення змін у REST API v1 та переходу на нові версії використовуються наступні підходи: * Blue-Green Deployment: розгортання нової версії поруч із поточною з подальшим перемиканням трафіку на рівні Load Balancer. * Версіонування: підтримка застарілих ендпоінтів (/v1/) протягом перехідного періоду для забезпечення зворотної сумісності з мобільними застосунками.

    3. Обробка технічного боргу (Technical Debt Management)

    З метою запобігання деградації архітектури встановлено такі правила: * Рефакторинг: кожен спринт виділяється 20% часу на виправлення застарілого коду та оптимізацію запитів до PostgreSQL. * Аналіз логів: регулярна перевірка Error Logs у системі моніторингу для виявлення прихованих вразливостей або вузьких місць продуктивності.

    4. Резервне копіювання та відновлення

    • Backups: автоматичне створення знімків бази даних кожні 24 години.
    • Відновлення: регулярна перевірка процедур відновлення даних з архівів для гарантії цілісності інформації про квитки та замовлення.

    5. Моніторинг та підтримка

    Система використовує Health Check ендпоінти для кожного мікросервісу. У разі виникнення критичних помилок (5xx) повідомлення автоматично надсилаються черговому інженеру через Notification Service.
